# Тестовый агент

В данной папке размещён тестовый прототип AI-агента, работающего с инструментами через **Model Context Protocol**. Прототип предназначен для проверки взаимодействия языковой модели с внешними MCP-серверами, в частности с сервером языка Pascal/Delphi.

Агент реализован на Python с использованием **LangGraph**, **LangChain MCP Adapters** и **GigaChat**. Он запускается в интерактивном режиме, принимает запросы пользователя из консоли, при необходимости вызывает доступные MCP-инструменты и выводит итоговый ответ.

## Структура папки

* `work/` — исходные файлы тестового агента.
* `work/agent.py` — основной файл запуска агента.
* `work/config.json` — конфигурация MCP-серверов.
* `work/requirements.txt` — список Python-зависимостей.
* `work/.env` — локальные параметры подключения к языковой модели.

## Назначение прототипа

Прототип предназначен для проверки схемы, при которой языковая модель получает доступ к внешним инструментам через MCP и может использовать их при ответе на пользовательские вопросы.

В текущей версии агент подключается к MCP-серверу `pasls`, который работает через `mcp-language-server` и Pascal language server. Это позволяет агенту обращаться к инструментам анализа Pascal/Delphi-проекта.

Основные задачи прототипа:

* загрузка конфигурации MCP-серверов из JSON-файла;
* подключение к MCP-серверам;
* получение списка доступных инструментов;
* создание ReAct-агента на основе LangGraph;
* вызов инструментов при необходимости;
* логирование tool-call'ов;
* измерение времени выполнения запроса;
* работа в интерактивном консольном режиме.

## Установка зависимостей

Перед запуском рекомендуется создать виртуальное окружение.

Пример для Windows Git Bash:

```bash
cd prototype-agent/work
python -m venv .venv
source .venv/Scripts/activate
pip install -r requirements.txt
```

Для Linux/macOS активация окружения будет выглядеть так:

```bash
source .venv/bin/activate
```

## Настройка `.env`

Файл `.env` содержит параметры подключения к языковой модели.

В текущей версии `agent.py` использует GigaChat и читает следующие переменные:

```text
CREDENTIALS_M2
SCOPE_M2
MODEL_M2
VERIFY_SSL_CERTS_M2
TEMPERATURE
```

Также в `.env` могут присутствовать переменные для другого провайдера:

```text
MODEL_M1
OPENROUTER_API_KEY_M1
OPENAI_API_BASE_M1
```

В текущей версии кода эти переменные не используются. Они могут быть оставлены как задел для последующего расширения, но для запуска текущего агента важны именно параметры GigaChat.

## Настройка `config.json`

Файл `config.json` содержит описание MCP-серверов, к которым подключается агент.

В текущей версии используется сервер:

```json
{
  "mcpServers": {
    "pasls": {
      "command": "C:/Users/Danii/go/bin/mcp-language-server.exe",
      "args": [
        "-workspace",
        "D:/Works/учёба/вкр/GpDelphiUnits_mini/src",
        "-lsp",
        "D:/Works/учёба/вкр/code/tools/pasls/pasls.exe"
      ],
      "transport": "stdio"
    }
  }
}
```

Перед запуском из нового расположения необходимо проверить и изменить пути:

* `command` — путь к `mcp-language-server.exe`;
* значение после `-workspace` — путь к Pascal/Delphi-проекту, который нужно анализировать;
* значение после `-lsp` — путь к `pasls.exe`.

Если проект был перенесён в другую папку, старые абсолютные пути нужно заменить на актуальные.

## Запуск агента

После установки зависимостей и настройки `.env` и `config.json` агент запускается командой:

```bash
python agent.py
```

После запуска появится консольный ввод:

```text
пользователь:
```

Можно вводить вопросы или команды. Для завершения работы используются команды:

```text
exit
quit
выход
```

## Логика работы

При запуске `agent.py` выполняет следующие действия:

1. Загружает переменные окружения из `.env`.
2. Читает MCP-конфигурацию из `config.json`.
3. Создаёт клиент `MultiServerMCPClient`.
4. Открывает постоянные MCP-сессии через `AsyncExitStack`.
5. Получает список инструментов с каждого MCP-сервера.
6. Оборачивает инструменты в логирующую оболочку.
7. Создаёт ReAct-агента через `create_react_agent`.
8. Запускает интерактивный цикл общения с пользователем.

Для одной интерактивной сессии используется общий `thread_id`, поэтому агент сохраняет контекст диалога через `InMemorySaver`.

## Логирование инструментов

В прототипе реализовано дополнительное логирование MCP-инструментов.

Для каждого tool-call выводится:

* номер вызова инструмента;
* время до первого tool-call;
* имя инструмента;
* входные параметры инструмента;
* длительность выполнения;
* ошибка, если инструмент завершился неуспешно.

После обработки пользовательского запроса агент выводит:

* общее время выполнения запроса;
* количество вызовов инструментов;
* время после первого вызова инструмента до завершения ответа;
* краткие ответы инструментов;
* итоговый ответ агента.

Такая информация полезна для отладки и анализа поведения агента.

## Основные файлы

### `agent.py`

Главный файл прототипа.

Содержит:

* создание GigaChat-модели;
* загрузку MCP-конфига;
* подключение к MCP-серверам;
* получение MCP-инструментов;
* обёртку инструментов для логирования;
* создание LangGraph ReAct-агента;
* интерактивный консольный цикл.

### `config.json`

Файл конфигурации MCP-серверов.

В текущей версии описывает подключение к Pascal language server через `mcp-language-server`.

### `requirements.txt`

Содержит Python-зависимости:

* `langchain-mcp-adapters`;
* `langgraph`;
* `langchain-core`;
* `langchain-gigachat`;
* `python-dotenv`.

### `.env`

Содержит локальные параметры подключения к языковой модели и настройки генерации.

## Итог

Тестовый агент демонстрирует базовый механизм подключения LLM-агента к внешним инструментам через MCP. Он показывает, как языковая модель может получать список доступных инструментов, вызывать их при необходимости, работать с результатами tool-call'ов и сохранять контекст интерактивной сессии.
