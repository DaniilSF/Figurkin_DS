# Учебные и выпускные работы

Репозиторий содержит материалы по учебным практикам, курсовой работе, выпускной квалификационной работе и тестовому прототипу AI-агента.

Материалы сгруппированы по отдельным разделам. В каждом разделе находится собственный `README.md` с описанием содержимого, назначения работы, структуры файлов и особенностей запуска программной части.

## Структура репозитория

```text
.
├── practice-3-course/
├── practice-4-course/
├── course-work/
├── diplom/
└── prototype-agent/
```

## Разделы репозитория

### `practice-3-course/`

Материалы практики за 3 курс.

В разделе размещён проект **Dispatcher** — клиент-серверное веб-приложение для просмотра справочной информации, связанной с диспетчеризацией перевозок, оборудованием, местами погрузки и разгрузки, расстояниями, типами грузов и отгрузками.

Проект реализован с использованием:

* FastAPI;
* Vue 3;
* PostgreSQL;
* Docker Compose.

Основная программная часть находится в папке:

```text
practice-3-course/work/
```

### `practice-4-course/`

Материалы практики за 4 курс.

В разделе размещён прототип веб-приложения для управления проектами изображений и ручной разметки объектов на изображениях. Пользователь может создавать проекты, загружать изображения, выделять прямоугольные области и задавать классы объектов.

Проект реализован с использованием:

* FastAPI;
* Vue 3;
* PostgreSQL;
* Docker Compose.

Основная программная часть находится в папке:

```text
practice-4-course/work/
```

### `course-work/`

Материалы междисциплинарной курсовой работы на тему: **«Разработка компонентов информационной системы для исследовательского поиска в массивах структурированных данных, взаимодействующих с большими языковыми моделями»**.

Курсовая работа посвящена разработке прототипа программы-аннотатора Python-кода. Аннотатор анализирует LLM-зависимые приложения, находит фрагменты интеграции с конкретным провайдером языковой модели и добавляет комментарии-подсказки для последующей миграции.

Структура раздела:

```text
course-work/
├── docs/
├── presentation/
└── work/
```

Назначение папок:

* `docs/` — текст курсовой работы в редактируемом и PDF-формате;
* `presentation/` — презентация и материалы для выступления;
* `work/` — программная часть, правила аннотирования, тестовые проекты и результаты экспериментов.

### `diplom/`

Материалы выпускной квалификационной работы на тему: **«Разработка информационной системы для исследовательского поиска в массивах структурированных данных с помощью больших языковых моделей»**.

ВКР посвящена разработке локальной информационной системы для анализа Pascal/Delphi-кодовой базы. Система формирует Markdown-wiki по исходному коду, индексирует её в RAG-хранилище и предоставляет AI-агенту инструмент поиска по структуре проекта.

Структура раздела:

```text
diplom/
├── docs/
├── presentation/
└── work/
```

Назначение папок:

* `docs/` — текст выпускной квалификационной работы;
* `presentation/` — презентация к защите;
* `work/` — программная часть ВКР.

Программная часть включает:

* генератор Markdown-wiki по Pascal/Delphi-коду;
* pipeline сборки wiki и RAG-индекса;
* MCP-wrapper для подключения RAG-поиска к AI-агенту;
* конфигурационные файлы;
* описание компонентов проекта.

Перед запуском программной части ВКР необходимо проверить локальные пути в конфигурационных файлах. Подробности указаны в `diplom/README.md`.

### `prototype-agent/`

Тестовый прототип AI-агента, работающего с инструментами через **Model Context Protocol**.

Агент реализован на Python с использованием LangGraph, LangChain MCP Adapters и GigaChat. Он запускается в интерактивном консольном режиме, подключается к MCP-серверам, получает список доступных инструментов и вызывает их при необходимости.

Основная программная часть находится в папке:

```text
prototype-agent/work/
```

Перед запуском необходимо проверить абсолютные пути в `prototype-agent/work/config.json`. Подробности указаны в `prototype-agent/README.md`.
