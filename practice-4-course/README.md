# Практика 4 курса

В данной папке размещены материалы практики за 4 курс. Работа представляет собой веб-приложение для управления проектами изображений и ручной разметки объектов на изображениях с помощью прямоугольных областей.

Проект реализован как клиент-серверное приложение. Серверная часть разработана на **FastAPI** и отвечает за хранение проектов, изображений и аннотаций. Клиентская часть разработана на **Vue 3** и предоставляет интерфейс для создания проектов, загрузки изображений, просмотра изображений и разметки объектов. В качестве базы данных используется **PostgreSQL**. Запуск всех компонентов выполняется через **Docker Compose**.

## Структура папки

* `work/` — исходные файлы проекта практики.
* `work/backend/` — серверная часть приложения на FastAPI.
* `work/frontend/` — клиентская часть приложения на Vue 3.
* `work/docker-compose.yml` — конфигурация запуска базы данных, backend и frontend.
* `work/README.md` — исходное краткое описание проекта.

## Назначение программы

Программа предназначена для организации набора изображений по проектам и выполнения ручной разметки объектов на изображениях. Пользователь может создать проект, загрузить в него изображения, открыть нужное изображение и выделить на нём прямоугольные области. Для каждой области задаётся класс объекта.

Такая логика может использоваться как прототип системы подготовки датасетов для задач компьютерного зрения. В текущей реализации также присутствует имитация ML-предсказания, которая создаёт случайные прямоугольные области на изображении.

## Запуск проекта

Для запуска требуется установленный Docker и Docker Compose.

Перейдите в папку проекта:

```bash
cd practice-4-course/work
```

Запустите проект:

```bash
docker compose up --build
```

После запуска будут доступны следующие сервисы:

* frontend: `http://localhost:8080`;
* backend API: `http://localhost:8000`;
* Swagger-документация API: `http://localhost:8000/docs`;
* PostgreSQL: `localhost:5432`.

Остановить проект можно командой:

```bash
docker compose down
```

Если нужно удалить сохранённые данные базы и загруженные изображения, можно выполнить:

```bash
docker compose down -v
```

## Общая архитектура

Проект состоит из трёх основных частей:

1. **База данных PostgreSQL**
   Используется для хранения проектов, метаданных изображений и аннотаций.

2. **Backend на FastAPI**
   Реализует REST API, работает с базой данных, сохраняет загруженные изображения и отдаёт их через публичные ссылки.

3. **Frontend на Vue 3**
   Предоставляет пользовательский интерфейс для работы с проектами, изображениями и разметкой.

Компоненты запускаются через `docker-compose.yml`. Отдельные Docker-контейнеры используются для базы данных, backend и frontend.

## Backend

Серверная часть находится в папке `work/backend`.

Основные файлы backend:

* `Dockerfile` — описание сборки backend-контейнера;
* `requirements.txt` — список Python-зависимостей;
* `app/main.py` — основной файл FastAPI-приложения;
* `app/db.py` — настройка подключения к базе данных;
* `app/models.py` — ORM-модели SQLAlchemy;
* `app/schemas.py` — Pydantic-схемы входных и выходных данных;
* `app/storage.py` — функции сохранения и удаления загруженных файлов.

### Логика backend

В `main.py` создаётся FastAPI-приложение, настраивается CORS, подключается статическая раздача загруженных файлов и регистрируются API-маршруты.

При запуске приложения выполняется инициализация базы данных. Для этого используется функция, которая несколько раз пытается подключиться к PostgreSQL и создать таблицы через SQLAlchemy. Это нужно, потому что backend может стартовать раньше, чем база данных полностью готова к работе.

Основные API-маршруты:

* `GET /health` — проверка работоспособности backend;
* `GET /projects` — получение списка проектов;
* `POST /projects` — создание проекта;
* `DELETE /projects/{id}` — удаление проекта вместе с его изображениями;
* `GET /projects/{id}/images` — получение изображений выбранного проекта;
* `POST /projects/{id}/images` — загрузка изображений в проект;
* `DELETE /images/{image_id}` — удаление изображения;
* `GET /images/{image_id}/annotations` — получение аннотаций изображения;
* `POST /images/{image_id}/annotations` — сохранение аннотаций изображения;
* `DELETE /annotations/{annotation_id}` — удаление отдельной аннотации;
* `POST /predict/{image_id}` — имитация ML-предсказания для изображения.

Файл `storage.py` выделяет работу с файловым хранилищем в отдельный слой. Сейчас изображения сохраняются локально в папку, подключённую как Docker volume. При необходимости этот слой можно заменить на работу с S3 или MinIO без полной переработки API.

## База данных

В проекте используются три основные сущности:

* `Project` — проект разметки;
* `Image` — изображение, загруженное в проект;
* `Annotation` — прямоугольная область разметки на изображении.

Модель `Project` хранит название проекта и дату создания. Один проект может содержать несколько изображений.

Модель `Image` хранит связь с проектом, исходное имя файла, MIME-тип, размер файла, имя файла в хранилище и дату загрузки.

Модель `Annotation` хранит координаты прямоугольной области, ширину, высоту, имя класса и связь с изображением.

Координаты аннотаций сохраняются в относительном виде. Это означает, что значения `x`, `y`, `w` и `h` задаются не в пикселях, а как доли от размеров изображения. Такой подход позволяет корректно отображать разметку при изменении масштаба изображения.

## Frontend

Клиентская часть находится в папке `work/frontend`.

Основные файлы frontend:

* `Dockerfile` — сборка Vue-приложения и запуск через Nginx;
* `nginx.conf` — конфигурация Nginx для одностраничного приложения;
* `package.json` — зависимости и команды проекта;
* `src/main.js` — точка входа Vue-приложения;
* `src/App.vue` — корневой компонент;
* `src/router/index.js` — описание маршрутов;
* `src/services/api.js` — функции обращения к backend API;
* `src/pages/ProjectsPage.vue` — страница списка проектов;
* `src/pages/ProjectImagesPage.vue` — страница изображений проекта;
* `src/pages/ImageViewerPage.vue` — страница просмотра и разметки изображения.

### Логика frontend

Frontend реализован как одностраничное приложение на Vue 3. Навигация выполняется через Vue Router.

На странице проектов пользователь может создать новый проект, просмотреть список уже созданных проектов и удалить ненужный проект.

На странице выбранного проекта пользователь может загрузить изображения. Поддерживаются два способа загрузки:

* выбор файлов через стандартное окно;
* перетаскивание изображений в область загрузки.

После загрузки изображения отображаются в виде миниатюр. Пользователь может открыть конкретное изображение для просмотра и разметки.

На странице изображения реализована ручная разметка. Пользователь зажимает левую кнопку мыши и рисует прямоугольную область. После завершения рисования открывается форма, где нужно указать класс объекта. Созданная область сохраняется через backend API.

Также поддерживается редактирование существующей области. При клике по прямоугольнику можно изменить его класс или удалить область. При наведении на прямоугольник отображается подсказка с названием класса.

Для удобства реализовано масштабирование изображения от `1x` до `3x`. Разметка хранится в относительных координатах, поэтому прямоугольники остаются привязанными к нужным местам изображения при изменении масштаба.

Кнопка `ML Predict` вызывает backend-маршрут `/predict/{image_id}`. В текущей версии это не настоящая модель машинного обучения, а имитация: backend создаёт несколько случайных прямоугольников с классами вида `ML_obj_1`, `ML_obj_2` и так далее. Эта функция показывает, как в будущем можно подключить реальную модель распознавания объектов.

## Docker Compose

Файл `docker-compose.yml` запускает три сервиса:

* `db` — база данных PostgreSQL;
* `backend` — FastAPI-приложение;
* `frontend` — собранное Vue-приложение, отдаваемое через Nginx.

Для хранения данных используются Docker volumes:

* `pm_db_data` — данные PostgreSQL;
* `pm_uploads` — загруженные изображения.

Благодаря этому данные не пропадают при обычной остановке контейнеров через `docker compose down`.
