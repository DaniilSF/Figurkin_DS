# Курсовая работа

В данной папке размещены материалы междисциплинарной курсовой работы на тему: **«Разработка компонентов информационной системы для исследовательского поиска в массивах структурированных данных, взаимодействующих с большими языковыми моделями»**.

Работа посвящена задаче анализа и частичной подготовки Python-приложений, использующих большие языковые модели, к миграции между LLM-провайдерами. В качестве исходной проблемы рассматривается зависимость программных проектов от конкретного API или SDK, например OpenAI. При переходе на альтернативные решения, такие как GigaChat или YandexGPT, требуется находить в коде точки привязки к провайдеру: импорты, ключи доступа, создание клиента, вызовы модели и обработку ответа.

В рамках курсовой работы разработан прототип программы-аннотатора. Он анализирует Python-код по набору правил, находит характерные фрагменты интеграции LLM и вставляет рядом с ними комментарии-подсказки с предлагаемыми изменениями. Дополнительно формируется отчёт о сработавших правилах.

## Назначение программы-аннотатора

Аннотатор предназначен для первичного анализа Python-кода LLM-зависимых приложений. Он не выполняет полноценную автоматическую миграцию проекта, а помогает разработчику быстрее найти места, которые требуют изменения при переходе на другого провайдера языковой модели.

Программа решает следующие задачи:

* загружает набор правил из JSON-файла;
* анализирует исходный Python-файл;
* ищет совпадения по токенизированным шаблонам;
* поддерживает плейсхолдеры для обобщения имён переменных;
* вставляет комментарии с подсказками рядом с найденными фрагментами;
* формирует отчёт о применённых правилах.

## Формат правил

Правила задаются в JSON-формате. Каждое правило содержит:

* `id` — идентификатор правила;
* `pattern_type` — тип найденного фрагмента, например `imports`, `ui`, `auth`, `llm_init`;
* `comment` — пояснение назначения правила;
* `find` — шаблон кода, который нужно найти;
* `insert` — описание вставки, включая место вставки и текст комментария.

В шаблонах поддерживаются плейсхолдеры вида `$KEY`, `$LLM`, `$INPUT`. Они позволяют искать не только полностью одинаковые строки, но и похожие конструкции с разными именами переменных.

## Логика работы аннотатора

Работа программы строится следующим образом.

Сначала аннотатор загружает JSON-файл с правилами и исходный Python-файл. Затем исходный код токенизируется с помощью стандартного модуля `tokenize`. При поиске игнорируются комментарии, переносы строк, отступы и другие служебные элементы, которые не влияют на основную структуру проверяемого выражения.

После этого программа последовательно обрабатывает правила. Для каждого правила формируется токенизированный шаблон. Если правило содержит плейсхолдер, то на его месте допускается любое имя переменной. При нахождении совпадения программа определяет диапазон строк, формирует блок комментариев и подготавливает операцию вставки.

Вставки применяются снизу вверх, чтобы номера строк не смещались во время изменения файла. В результате создаётся новый Python-файл с комментариями-подсказками. Дополнительно создаётся текстовый отчёт, где указано, какие правила сработали, сколько было вставок и на каких строках были найдены совпадения.

## Запуск аннотатора

Для запуска через Docker нужно перейти в папку программной части:

```bash
cd course-work/work
```

Собрать Docker-образ:

```bash
docker build -t project-updater .
```

Запустить аннотатор:

```bash
docker run --rm -v "${PWD}:/data" -w /data project-updater rules_yandex.json Chat_with_search.py output.py
```

После выполнения будут сформированы:

* `output.py` — аннотированный файл;
* `report.txt` — отчёт о сработавших правилах.

## Запуск обработанного Streamlit-приложения

В папке `modified_program/` находится отдельный Dockerfile для запуска обработанных Streamlit-приложений.

Перейдите в папку:

```bash
cd course-work/work/modified_program
```

Соберите образ:

```bash
docker build -f LLM_Dockerfile -t my-streamlit .
```

Запустите нужный файл:

```bash
docker run --rm -p 8501:8501 -v "${PWD}:/app" my-streamlit Chat_with_search_output.py
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8501
```

## Итог

В результате курсовой работы разработан прототип инструмента, который помогает исследовать LLM-зависимый код и находить места, требующие изменения при миграции между провайдерами языковых моделей. Решение демонстрирует подход к исследовательскому поиску по структурированным данным, где в качестве анализируемого массива выступает исходный код Python-приложений.
