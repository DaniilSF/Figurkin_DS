# Практика 3 курса

В данной папке размещены материалы практики за 3 курс. Работа представляет собой веб-приложение **Dispatcher**, предназначенное для просмотра справочной информации, связанной с диспетчеризацией перевозок, оборудованием, местами погрузки и разгрузки, расстояниями, типами грузов и отгрузками.

Проект реализован как клиент-серверное приложение. Серверная часть разработана на **FastAPI** и предоставляет REST API для получения данных из базы. Клиентская часть разработана на **Vue 3** и отображает данные в виде таблиц с возможностью поиска. В качестве СУБД используется **PostgreSQL**. Развёртывание проекта выполняется через **Docker Compose**.

## Структура папки

* `work/` — исходные файлы проекта практики.
* `work/Backend/` — серверная часть приложения на FastAPI.
* `work/frontend/` — клиентская часть приложения на Vue 3.
* `work/init.sql` — SQL-скрипт создания и начального заполнения базы данных.
* `work/docker-compose.yml` — конфигурация запуска базы данных, backend и frontend.
* `work/.env` и `work/.env.example` — файлы с параметрами подключения к базе данных.
* `work/.gitlab-ci.yml` — конфигурация CI/CD для GitLab.

## Назначение программы

Приложение предназначено для отображения справочных и операционных данных, связанных с диспетчеризацией перевозок. Пользователь может просматривать:

* список оборудования;
* места погрузки и разгрузки;
* расстояния между пунктами;
* характеристики кузовов техники;
* типы перевозимых грузов;
* плановые и фактические отгрузки.

Интерфейс работает в режиме просмотра данных. Для удобства таблицы поддерживают поиск по основным полям.

## Запуск проекта

Для запуска требуется установленный Docker и Docker Compose.

Перейдите в папку проекта:

```bash
cd practice-3-course/work
```

Запустите сборку и запуск контейнеров:

```bash
docker compose up --build
```

После запуска будут доступны следующие сервисы:

* frontend: `http://localhost:5174`;
* backend API: `http://localhost:8001`;
* документация API Swagger: `http://localhost:8001/docs`;
* PostgreSQL: `localhost:5433`.

Остановить приложение можно командой:

```bash
docker compose down
```

Если необходимо удалить данные PostgreSQL, сохранённые в Docker volume, можно использовать:

```bash
docker compose down -v
```

## Общая архитектура

Проект состоит из трёх основных частей:

1. **База данных PostgreSQL**
   Хранит справочные и операционные данные. Структура базы создаётся с помощью файла `init.sql`.

2. **Backend на FastAPI**
   Отвечает за подключение к базе данных, получение записей и выдачу данных через REST API.

3. **Frontend на Vue 3**
   Получает данные с backend и отображает их пользователю в виде таблиц.

Связь между сервисами организована через Docker Compose. Frontend работает через Nginx, который также проксирует API-запросы к backend-сервису.

## База данных

В базе данных используются следующие таблицы:

* `equipment` — оборудование;
* `locations` — места погрузки и разгрузки;
* `distances` — расстояния между местами;
* `body_weights` — характеристики массы кузова оборудования;
* `cargo_types` — типы грузов;
* `shipments` — отгрузки.

Файл `init.sql` создаёт структуру таблиц и заполняет их тестовыми данными. Между таблицами настроены связи через внешние ключи. Например, отгрузка связана с оборудованием, типом груза, местом погрузки и местом разгрузки.

## Backend

Серверная часть находится в папке `work/Backend`.

Основные файлы backend:

* `Dockerfile` — описание сборки backend-контейнера;
* `requirements.txt` — список Python-зависимостей;
* `App/main.py` — точка входа FastAPI-приложения;
* `App/database.py` — настройка подключения к базе данных;
* `App/models.py` — ORM-модели SQLAlchemy;
* `App/schemas.py` — Pydantic-схемы для формирования ответов API;
* `App/crud.py` — функции получения данных из базы;
* `App/routers/` — набор маршрутов API.

### Логика backend

В `main.py` создаётся объект FastAPI-приложения, подключается CORS и регистрируются маршруты для основных сущностей:

* `/equipment`;
* `/locations`;
* `/distances`;
* `/body_weights`;
* `/cargo_types`;
* `/shipments`.

Файл `database.py` отвечает за чтение строки подключения из `.env`, создание асинхронного SQLAlchemy engine и выдачу сессии базы данных через зависимость `get_async_session`.

Файл `models.py` описывает таблицы базы данных как ORM-классы. В моделях также задаются связи между сущностями, например связь отгрузки с оборудованием, типом груза и локациями.

Файл `schemas.py` задаёт структуры данных, которые возвращаются через API. Для связанных сущностей используются вложенные схемы, поэтому на frontend передаются не только идентификаторы, но и связанные объекты. Например, в отгрузках отображается название оборудования и название типа груза.

Файл `crud.py` содержит функции выборки данных. Для связанных данных используется `selectinload`, чтобы вместе с основной записью загрузить связанные сущности и корректно отобразить их на frontend.

Папка `routers/` содержит отдельные маршрутизаторы для каждой сущности. Каждый маршрутизатор предоставляет методы получения списка записей и получения записи по идентификатору.

## Frontend

Клиентская часть находится в папке `work/frontend`.

Основные файлы frontend:

* `Dockerfile` — сборка Vue-приложения и запуск через Nginx;
* `nginx.conf` — конфигурация Nginx и проксирование API-запросов;
* `package.json` — зависимости и команды frontend-проекта;
* `src/main.js` — точка входа Vue-приложения;
* `src/App.vue` — корневой компонент;
* `src/router/index.js` — маршруты страниц;
* `src/layouts/MainLayout.vue` — общий макет с навигацией;
* `src/components/TableList.vue` — универсальный компонент таблицы;
* `src/pages/` — страницы для отображения отдельных сущностей.

### Логика frontend

Frontend реализован как одностраничное приложение. Навигация между разделами выполняется через Vue Router.

В `MainLayout.vue` задаётся верхнее меню со ссылками на основные разделы:

* оборудование;
* места;
* расстояния;
* веса кузовов;
* типы груза;
* отгрузки.

Для отображения данных используется общий компонент `TableList.vue`. Он получает массив строк, описание колонок и список полей для поиска. Благодаря этому один и тот же компонент используется для разных таблиц.

В компоненте `TableList.vue` реализована фильтрация данных. Пользователь вводит строку поиска, после чего таблица показывает только те записи, где найдено совпадение в указанных полях. Для обращения к вложенным полям используется `lodash/get`, что позволяет отображать значения вида `equipment.name`, `cargo_type.name`, `from_location.name`.

Каждая страница в папке `src/pages/` загружает данные с соответствующего API-маршрута и передаёт их в `TableList`.

## Docker Compose

Файл `docker-compose.yml` описывает запуск трёх сервисов:

* `db` — контейнер PostgreSQL;
* `backend` — контейнер FastAPI-приложения;
* `frontend` — контейнер Vue-приложения, собранного и запущенного через Nginx.

При запуске PostgreSQL автоматически выполняет файл `init.sql`, который создаёт таблицы и добавляет тестовые данные.

Backend подключается к базе данных по строке подключения из `.env`.

Frontend доступен на внешнем порту `5174`. Backend доступен на внешнем порту `8001`. База данных доступна на внешнем порту `5433`.

## Итог

В результате практики разработано контейнеризированное веб-приложение для просмотра данных диспетчерской системы. Проект демонстрирует работу с базой данных PostgreSQL, создание REST API на FastAPI, отображение данных во Vue-интерфейсе и запуск всех компонентов через Docker Compose.
