# Выпускная квалификационная работа

В данной папке размещены материалы выпускной квалификационной работы на тему: **«Разработка информационной системы для исследовательского поиска в массивах структурированных данных с помощью больших языковых моделей»**.

Работа посвящена разработке локальной информационной системы для анализа и навигации по Pascal/Delphi-кодовой базе. Система преобразует исходный код проекта в Markdown-wiki, пригодную для просмотра в Obsidian, индексирует полученную документацию в RAG-хранилище и предоставляет AI-агенту инструмент поиска по структуре проекта.

Основная идея системы заключается в создании промежуточного навигационно-поискового слоя между исходным кодом и разработчиком. Такой слой помогает быстрее находить модули, классы, методы и связи между сущностями проекта без полного ручного просмотра всей кодовой базы.

## Структура папки

* `docs/` — текст выпускной квалификационной работы.
* `presentation/` — презентация к защите выпускной квалификационной работы.
* `work/` — программная часть ВКР.

## Документы

В папке `docs/` размещается текст ВКР:

* `diplom.docx` — редактируемая версия текста ВКР;
* `diplom.pdf` — PDF-версия текста ВКР.

## Презентация

В папке `presentation/` размещаются материалы для защиты:

* `diplom-presentation.pptx` — редактируемая презентация;
* `diplom-presentation.pdf` — PDF-версия презентации.

## Программная часть

Программная часть находится в папке `work/`.

Основные элементы:

* `.config/` — конфигурационные файлы для запуска pipeline, настройки путей, AI-провайдера и RAG-компонентов;
* `AGENTS.md` — инструкция для OpenCode-агента по работе с Pascal/Delphi-проектом, Markdown-wiki и RAG-поиском;
* `PROJECT_COMPONENTS_DESCRIPTION.md` — подробное описание устройства программной части;
* `README.md` — краткое описание программы;
* `code/` — собственный код проекта;
* `data/` — входные и выходные данные, логи и сгенерированная wiki;
* `tools/` — внешние инструменты и вспомогательные компоненты.

## Назначение программной системы

Программная система предназначена для исследовательского поиска по Pascal/Delphi-кодовой базе с использованием больших языковых моделей и RAG-подхода.

Система выполняет следующие задачи:

* получает на вход Pascal/Delphi-проект;
* извлекает структуру исходного кода;
* формирует Markdown-wiki по unit, классам и методам;
* дополняет wiki описаниями, полученными с помощью языковой модели;
* подготавливает wiki для просмотра в Obsidian;
* индексирует Markdown-документы в RAG-хранилище;
* предоставляет OpenCode-агенту MCP-инструмент для поиска по wiki;
* помогает разработчику получать ответы о структуре и назначении элементов проекта.

## Общая схема работы

Основной маршрут обработки данных:

```text
Pascal/Delphi source
→ PasDoc SimpleXML
→ Markdown wiki generator
→ Obsidian-compatible Markdown wiki
→ markdown-rag-mcp index
→ Milvus vector database
→ markdown_rag_wrapper MCP
→ OpenCode agent
```

Иными словами, пользователь помещает Pascal/Delphi-проект в рабочую область, запускает pipeline, получает Markdown-wiki и индекс для поиска. После этого AI-агент может использовать RAG-поиск для нахождения релевантных сущностей проекта и ответа на вопросы разработчика.

## Папка `code`

Папка `code/` содержит собственные компоненты программной системы.

Основные подпапки:

* `code/wiki/` — скрипт генерации Markdown-wiki по Pascal/Delphi-коду;
* `code/pipeline/` — скрипты сборки wiki и RAG-индекса;
* `code/mcp/` — MCP-wrapper для подключения RAG-поиска к OpenCode;
* `code/agent/` — фрагменты конфигурации агента.

### Wiki generator

Основной скрипт генерации wiki расположен в:

```text
code/wiki/ai-descr-ai-v3-config-rag-md-v3.sh
```

Он отвечает за обработку Pascal/Delphi-кода, запуск PasDoc, разбор SimpleXML и формирование Markdown-файлов. В результате создаются страницы по модулям, классам и методам проекта.

### Pipeline

Основной скрипт pipeline расположен в:

```text
code/pipeline/build-wiki-rag.sh
```

Он выполняет полный сценарий сборки:

1. читает локальную конфигурацию;
2. подготавливает рабочие папки;
3. запускает или проверяет Milvus;
4. вызывает генератор Markdown-wiki;
5. индексирует wiki через `markdown-rag-mcp`;
6. выполняет контрольный поиск;
7. сохраняет логи.

Для ручной проверки уже собранного RAG-индекса используется:

```text
code/pipeline/check-rag.sh
```

### MCP-wrapper

Wrapper расположен в:

```text
code/mcp/markdown-rag-mcp-wrapper/
```

Он предоставляет OpenCode-агенту инструменты:

* `markdown_rag_ping` — проверка доступности wrapper;
* `markdown_rag_warmup` — прогрев RAG backend;
* `markdown_rag_search` — поиск по Markdown RAG-индексу;
* `markdown_rag_reload_engine` — сброс кэшированного RAGEngine.

Wrapper не занимается генерацией wiki и индексацией. Эти задачи выполняются заранее через pipeline.

## Папка `data`

Папка `data/` предназначена для входных и выходных данных.

Основные подпапки:

* `data/input_project/` — место для исходного Pascal/Delphi-проекта;
* `data/wiki_out/` — результат генерации Markdown-wiki;
* `data/logs/` — логи выполнения pipeline;
* `data/cache/` — кэш описаний, если он используется при генерации.

В Obsidian обычно открывается папка:

```text
data/wiki_out/current
```

Она выступает как Obsidian vault и содержит сгенерированные Markdown-страницы.

## Папка `tools`

Папка `tools/` содержит внешние инструменты и backend-компоненты, необходимые для работы системы.

Основные подпапки:

* `tools/pasdoc/` — PasDoc для извлечения структуры Pascal/Delphi-кода;
* `tools/xmlstarlet/` — инструмент обработки XML;
* `tools/pasls/` — Pascal language server;
* `tools/bin/` — дополнительные бинарные файлы, например `jq.exe` и `mcp-language-server.exe`;
* `tools/markdown-rag-mcp/` — backend для индексации Markdown-документов и поиска по ним.

В некоторых папках могут находиться текстовые файлы-заглушки с пояснением, какие бинарные файлы должны быть размещены вручную.

## Конфигурация

Локальные настройки находятся в папке `.config/`.

Основные файлы:

* `rag-pipeline.local.conf` — настройки полного pipeline;
* `ai-descr.local.conf` — настройки AI-провайдера для генерации описаний;
* `ai-descr.openai-compatible.example.conf` — пример настройки OpenAI-compatible endpoint;
* `ai-descr.openrouter.example.conf` — пример настройки OpenRouter.

Файл `rag-pipeline.local.conf` содержит пути к корню проекта, исходному Pascal/Delphi-коду, инструментам, wrapper-у, логам и RAG-компонентам.

Перед запуском из нового расположения необходимо проверить и при необходимости изменить абсолютные пути в конфигурационных файлах.

## Запуск полного pipeline

Перед запуском необходимо подготовить:

* Pascal/Delphi-проект в `data/input_project/`;
* необходимые внешние инструменты в `tools/`;
* Docker для запуска Milvus;
* Python/uv для запуска `markdown-rag-mcp`;
* локальные конфигурационные файлы с актуальными путями и параметрами запуска.

Перед запуском необходимо проверить пути в следующих файлах:

* `.config/rag-pipeline.local.conf` — основной конфигурационный файл pipeline. В нём задаются пути к корню проекта, входному Pascal/Delphi-проекту, папке вывода wiki, логам, инструментам и RAG-компонентам.
* `.config/ai-descr.local.conf` — настройки AI-провайдера, используемого при генерации описаний.
* `code/agent/opencode.fragment_example.json` — пример фрагмента конфигурации OpenCode.
* `code/mcp/markdown-rag-mcp-wrapper/server.py` — содержит путь по умолчанию к backend-папке `markdown-rag-mcp`. Если путь не передаётся через переменную окружения `MARKDOWN_RAG_BACKEND_DIR`, wrapper будет использовать значение из этого файла.
* `code/wiki/ai-descr-ai-v3-config-rag-md-v3.sh` — содержит fallback-пути для standalone-запуска генератора wiki. При запуске через полный pipeline эти значения переопределяются конфигурацией, но при ручном запуске скрипта их также нужно проверить.

Пример запуска из папки `work`:

```bash
cd diplom/work
sh code/pipeline/build-wiki-rag.sh
```

После выполнения pipeline в `data/wiki_out/current` будет сформирована Markdown-wiki, а документы будут проиндексированы в RAG-хранилище.

Если проект запускается из нового расположения, например после переноса из папки `finalversion` в `diplom/work`, основное внимание нужно обратить на абсолютные пути. Все пути, указывающие на старое расположение проекта, должны быть заменены на актуальные.

## Проверка RAG-поиска

Для проверки уже созданного индекса используется скрипт:

```bash
cd diplom/work
sh code/pipeline/check-rag.sh FindBucket
```

Скрипт выполняет поиск по уже построенному индексу и не пересоздаёт wiki.

## Работа с OpenCode-агентом

Файл `AGENTS.md` содержит правила поведения агента при работе с Pascal/Delphi-проектом.

Агент должен:

* сначала использовать RAG-поиск по Markdown-wiki для понимания структуры проекта;
* при необходимости уточнять детали по исходному коду;
* не выдумывать отсутствующие детали реализации;
* отвечать на русском языке;
* указывать найденную сущность, unit, class или method, если это полезно.

Такой подход снижает вероятность случайных ответов и помогает агенту опираться на подготовленную документацию проекта.

## Итог

В результате ВКР разработана локальная система, объединяющая генерацию документации по Pascal/Delphi-коду, Markdown-wiki, RAG-индексацию и AI-агента. Система предназначена для ускорения анализа крупных или устаревших Pascal/Delphi-проектов и помогает разработчику быстрее находить нужные сущности, понимать структуру проекта и ориентироваться в кодовой базе.
